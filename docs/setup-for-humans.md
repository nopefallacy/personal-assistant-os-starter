# Setup For Humans

Use this if you want to set the system up with an AI agent, but you want the moving parts to make sense first.

## Step 1

Install:

- Obsidian
- one AI runtime: Claude Code, Codex, Gemini CLI, or another agent that can read files and edit markdown

## Step 2

Open this folder as an Obsidian vault.

Do not try to customize everything first.
The point of this starter is to let the AI do the personalization work with you.

## Step 3

Install the plugin stack from [plugin-stack.md](plugin-stack.md).

Keep it minimal.
You can always add more later.

## Step 4

Pick your runtime:

- Claude Code
- Codex
- Gemini CLI
- another agent

Then copy the matching prompt from [setup-for-ai.md](setup-for-ai.md).

If you use Claude Code and like slash commands, this starter also includes optional commands in `.claude/commands/`.
You can use those instead of pasting the longer setup prompt every time.

## Step 5

Let the agent onboard you.

It should:

1. ask a short set of high-signal questions
2. figure out what kind of life and work you actually have
3. decide which folders and workflows should exist
4. update the memory layer first
5. only add plugins, structures, and automation that you will really use

## What Good Setup Looks Like

At the end of onboarding:

- your preferences are saved
- your active projects are mapped
- your main workflows are visible
- the runtime you use has enough context to be useful
- the vault still feels small

If it feels like a template warehouse, the setup went wrong.
