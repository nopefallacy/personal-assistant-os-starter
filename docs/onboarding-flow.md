# Onboarding Flow

The onboarding flow should feel like a sharp strategist, not a form.

## Phase 1: Identity

Ask for:

- name
- what they do
- their main work areas right now

## Phase 2: Runtime + Tools

Ask for:

- which AI runtime they actually use
- whether they use Obsidian only, or Obsidian plus calendar, task tools, meetings, content systems, or coding tools

## Phase 3: Workflow Detection

Figure out whether they need:

- project tracking
- task management
- meetings
- content creation
- research workflows
- personal life tracking

## Phase 4: Recommendation

Recommend what to enable and what to skip.

The agent should explain:

- why each enabled area belongs
- why skipped areas are being left out for now

## Phase 5: Build

Only after the above:

- update `.ai/memory/`
- create the relevant folders and guide files
- wire runtime adapters only as needed

## Default Bias

Bias toward:

- fewer folders
- fewer prompts
- better naming
- clearer memory

Not toward:

- maximum features
- maximum templates
- maximum plugin count
